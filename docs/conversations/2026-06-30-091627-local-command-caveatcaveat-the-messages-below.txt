‚Źļ Why it was failing

  JSON mode worked fine ‚ÄĒ the model returned valid JSON every call. The problem
  was what it put in css: bare declarations like color: blue;, which are not a
  valid stylesheet rule, so applyCss injected them and nothing matched. Only the
  one time it happened to emit a full selector { ... } rule did anything
  change. Asking a model to reliably produce a full rule with a correct selector
  every time is exactly the kind of thing it'll get inconsistently.

  The fix

  Don't let the model own the selector ‚ÄĒ the extension already computed the
  exact one. Now:
  - Model returns { declarations: "color: blue;" } only (no selector, no
  braces).
  - Server wraps it: ${selector} { ${declarations} } ‚Üí always a valid,
  applicable rule.
  - A regex fallback strips braces if the model wraps anyway, so both cases
  normalize.

  This makes correct structure deterministic regardless of model whims.

  skipped: per-model retry on "JSON Mode couldn't be met" ‚ÄĒ add if you see 502s;
  the wrap already handles the common failure. Note your cssPath produces
  selector-only paths, so combine cleanly with the per-element edits map from
  earlier.
